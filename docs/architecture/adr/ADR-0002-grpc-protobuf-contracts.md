# ADR-0002: gRPC + Protobuf as internal contracts

## Статус
Accepted (2026-01-17)

## Контекст
Требуется быстрый и строгий межсервисный протокол, единые контракты, генерация клиентов/серверов.

## Решение
Внутри системы используем **gRPC**, контракты фиксируем в `invenlore/proto`.

## Последствия
+ Строгая типизация и совместимость
+ Генерация кода и документации
- Требуются правила версионирования и дисциплина изменений

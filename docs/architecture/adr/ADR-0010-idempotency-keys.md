# ADR-0010: Idempotency keys for write APIs

## Статус
Accepted (2026-01-17)

## Контекст
Клиенты и gateway могут ретраить запросы. Без идемпотентности возможны дубли ревизий/операций.

## Решение
Вводим `Idempotency-Key` для write запросов. Сервис хранит результат по ключу и возвращает его повторно.

## Последствия
+ Безопасные ретраи
- Нужны индексы/TTL на коллекции idempotency
